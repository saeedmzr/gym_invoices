Tests

Tests contains 3 diffrents test cases :

1 - Login Tests

2 - Membership Tests

3 - Club Tests



Login Tests

  • 1 - user can login with_correct credentials :

    This method test user login with correct credentials and return success if its correct.

  • 2 - user cannot login with incorrect password :

    This method with test user login with wrong credentials and return 422 status code.



Membership Tests

  • 1 - membership doesnt work if it pass expire time :

    In this method we will check if a membership pass expire time it doesnt work and user cannot use it.

  • 2 - membership doesnt work if it got canceled :

    This method will test membership that got canceled cant be used.

  • 3 - membership doesnt work if it doesnt have any credits:

    You can check-in to club as a membership ,if your membership as a account.this method will test that you cant use a membership that has zero credits.

  • 4 - membership work :

    This method will test a available membership with credits,and return status code 200.



Club Tests

  • 1 - check in normally should return status code 200 :

    This will test your normal check-in at a club .

  • 2 - invoice get create first time that user check in a club :

    Test invoice record will create successfully the first time you check-in at a club .

  • 3 - if user check-in in two different clubs then he got two invoice records:

    Invoice records are unique for user-club. for example if you check-in at 2 different clubs in a month, you got 2 invoice records.

  • 4 - user can check in with no credits of membership:

    If a user check-in with no credits in his membership account, he should check-in but he will check-in as a GUEST.

  • 6 - invoice total amount should return exactly what we except for guest:

    In this test we want to test a user that check-in as a club. first, we create a club and check in at it with a user 4 times. then we test the invoice amount should equal to cost_per_check_in from that club multiply 4.

  • 7 - invoice total amount should return exactly what we except for membership:

    If a user check-in with only membership status, then in the invoice amount would be equal to zero since he doesn't have to pay anything

  • 8 - invoice total amount should return exactly what we except for membership if it ends between:

    One of the most challenging parts of the product is when the user check-in with both statuses: membership and guest. for example, imagine a user has 2 credits in his membership account .then he can check-in 2 times with that. after that he wants to check-in 2 more times .but he doesn't have any credits left. the amount of invoice in this scenario should be the club's cost per check-in multiply 2.

  • 9 - invoice get created again if month passes:

    If you check-in after a month in any club then you have 2 records in the invoices table.

  • 10 - cant check in twice a day in same club:

    You can't check in at a certain club twice a day.
