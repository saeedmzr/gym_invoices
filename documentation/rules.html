Rules

Check-in Rules

  • 1 - You can check in once a day in a certain club. if you try to check in twice a day in a certain club it returns an error.
  • 2 - You can check in, whether you have a membership in that club or not. If you have a membership, after check-in your membership's credits decrease.otherwise, if you don't have a membership then you have to pay for that later. Every club has a property called "cost_per_check_in"
  • 3 - For every check-in, we have one record in the checkins table. checkins table contains invoice lines, since storing in JSON format is not a good idea
  • 4 - If you check-in as membership successfully, the invoice's amount doesn't change .but you can access that check-in data in the checkins table
  • 5 - Every record in the invoices table is one of the user's activities in a certain club. for example: if the user check-in two different clubs in December, then he has two records in the invoices table.
  • 6 - The invoice amount property is what the user has to pay. so, if the user check-in to a club with only membership status, the amount of invoice record is equal to zero.
  • 7 - If a user check-in with only membership status, then the invoice status would be Paid and the amount of it would be zero since he doesn't have to pay anything.
  • 8 - If a user check-in with membership and guest status, the status of the invoice record will be outstanding with the first guest check-in and the amount of it will be the sum of the user check-ins as a guest.
